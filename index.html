What is Express Rate Limiter?

Express Rate Limiter is a basic rate-limiting middleware for Express. Use it to limit repeated requests to public APIs and/or endpoints such as password reset. It helps protect your infrastructure from brute-force attacks and "noisy neighbor" scenarios.

Note: By default, the rate limiter uses an in-memory store. For production environments with multiple instances, we recommend using the Redis Store.

Quick Start

npm install express-rate-limit

Basic Implementation

To apply a global limit to all requests, use the following configuration:

JavaScript (Node.js) Copy 
const rateLimit = require('express-rate-limit');

const limiter = rateLimit({
    windowMs: 15 * 60 * 1000, // 15 minutes
    max: 100, // Limit each IP to 100 requests per window
    standardHeaders: true, // Return rate limit info in headers
    legacyHeaders: false, // Disable the `X-RateLimit-*` headers
});

// Apply to all requests
app.use(limiter);

Configuration Parameters

Parameter Default Description
windowMs 60000 Time window in milliseconds.
max 5 Maximum number of connections per IP per window.
message "Too many..." The error message sent to the client.
